TT - Testing Tool Documentation

Requirements:

A copy of ade must be found in the current PATH.

Non-execution operations can still be performed (show, dry-run) without a copy of ade. tt will run against the ade in the current path - easy to switch testing directories!

Packages vs. Groups

Packages
  • a specific physical subdirectory
  • contains one or more tests in the tt-defined format.
  • tt-defined format: any directory containing test.exe and optionally test.attr.
  • For example, in the following directory structure: tests/sta/test1
    tests/sta/test2
    tests/sta/test3
    tests/sta/newtests/test1
    the package is sta. The tests within package sta are test1, test2, test3 and newtests/test1.
  • package names CANNOT include underscores

Goal: Every subdirectory within "src" that contains ade code will have a corresponding "package" subdirectory within the tests directory tree.

Groups
  • a virtual construct
  • relates tests from various packages
  • group tests have a common function
  • each test can belong to any number of groups (via test.attr)
  • e.g., sta/test1 and db/test2 belong to "sanity" group
    sta/test3 and db/test3 belong to "extended" group.
  • a new group can be created at any time by anyone
