TT - Testing Tool Documentation

run_test shortcut

  • run_test: a shortcut
  • automatically installed into "DIST" directory along with ade
  • requires ade DIST directory to be in PATH
  • runs tests from the test directory that belongs to THAT DIST directory
  • execute tests from ANY test package/group from any location
  • expected to be most common way to use tt

All commands remain the same as before, except that test_ is added automatically, so should be omitted.

For instance, if package sta is being worked on, after recompilation and installation:

run_test sta

will run all sta tests without leaving the current directory.


run_test has two special features:
  • puts the tt.log file into the current directory rather than the tt directory
  • the command "run_test template" creates a template test in the current directory.
    Creates (and overwrites): Makefile, test.exe, test.attr and tt.log.


At minimum, the user must still create/provide two more files:
test.log.GOLD
test.tcl
